Lions Creek Rewards - Lightsail quick rebuild and troubleshooting

Code refresh
-----------
git fetch --all --prune
git checkout main
git reset --hard origin/main
git log --oneline --decorate --graph --all

Service checks
--------------
# nginx errors
sudo tail -n 200 /var/log/nginx/error.log

# app service
sudo systemctl status lions-creek-rewards -l --no-pager
sudo journalctl -u lions-creek-rewards -n 200 -l --no-pager

# expiry timer + job
sudo systemctl status lions-creek-rewards-expire.timer -l --no-pager
sudo journalctl -u lions-creek-rewards-expire.service -n 100 -l --no-pager

# confirm app is listening on 3000 locally
sudo ss -ltnp | grep ':3000' || true

# bypass nginx and hit the node app directly
curl -sv http://127.0.0.1:3000/ -o /dev/null

Deploy using the repo helper
----------------------------
bash scripts/Server_Git_Code_Deploy.sh

Environment file
----------------
Production secrets should live in:

/etc/lions-creek-rewards/lions-creek-rewards.env

Check the effective systemd configuration:

sudo systemctl show lions-creek-rewards -p FragmentPath -p EnvironmentFiles -p ExecStart
sudo systemctl cat lions-creek-rewards

Manual expiry job test
----------------------
sudo bash -lc 'set -a; source /etc/lions-creek-rewards/lions-creek-rewards.env; set +a; curl -i -fsS -H "x-job-token: ${JOB_TOKEN}" "http://127.0.0.1:3000/jobs/expire?all=1"'

If the box becomes unhealthy
---------------------------
# check for OOM kills
dmesg | egrep -i 'killed process|oom' | tail -n 50

# stop stuck npm/node processes
sudo pkill -f npm || true
sudo pkill -f node || true

# check disk use
df -h
du -sh /var/www/lions-creek-rewards/node_modules 2>/dev/null
